## Codex pre-flip consult

**실행 시각**: 2026-05-21T (Phase A)  
**판정**: FAIL — HIGH 3건. 각 mitigation을 plan에 반영 후 Phase B 진입.

### Q1 (MEDIUM) — Gate bypass risk
**Finding**: Axis 1 bypass leaves rollout without adoption-diversity signal. Gate intent = prove flow works across enough distinct environments.  
**Rec**: Document as explicit data-gathering bypass. Do not declare "gate passed" until Axis 1 reaches threshold.  
**Mitigation adopted**: 이미 plan §Context에 명시. decision-log 엔트리에 owner 우회 사유 기재.

### Q2 (LOW) — Deploy window measurement censoring
**Finding**: Old/new build mixed window (~3-5min). Not a data integrity issue; measurement attribution issue.  
**Rec**: Exclude/annotate deploy propagation window; start T+ monitoring after redeploy stabilizes.  
**Mitigation adopted**: Phase G 모니터링을 redeploy Ready 확인 후 시작.

### Q3 (HIGH) — Telemetry suppression distorts axis2.failRate
**Finding**: Brave/uBlock users generate ghost `started` with no terminal event. `failRate = failed/(completed+failed)` excludes abandoned → suppressed users inflate abandonment silently without triggering rollback invariant #5.  
**Rec**: Compare `started` vs `completed + failed`. Treat high abandonment as denominator unreliability.  
**Mitigation adopted**: Phase G에 denominator reliability check 추가:  
  - If `started > (completed+failed) × 3` → high abandonment zone → `failRate` 평가 보류, 대신 `completed` 절대값 추세로 판단  
  - Invariant #5 기준: failRate ≤ 20% *AND* abandonment ratio acceptable

### Q4 (HIGH) — Rate-limit NAT collision risk
**Finding**: Limiter is per-IP only (not IP+UA). 5/min ceiling. NAT office → multiple users = same IP → 429 → legitimate users blocked.  
**Rec**: Monitor 429 responses on `/api/internal/issue-collector-token` during flip.  
**Mitigation adopted**: Phase G 모니터링 시그널에 429 rate 추가. 코드 변경 없음 (non-scope). 현 rate-limit은 flip 이전부터 적용된 기존 production 동작.

### Q5 (HIGH) — Kill-switch scope: "new exposure off" not "active sessions off"
**Finding**: Redeploy stops new page loads but already-loaded JS + valid tokens continue. In-flight `/api/burnindex` POSTs complete. Old tabs survive redeploy window.  
**Rec**: Define kill-switch as "new exposure off." For security rollback (secret leak), rotate COLLECTOR_HMAC_SECRET.  
**Mitigation adopted**: Invariant #1 (secret leak → rotate) already covers security rollback. Phase H에 명시: "rollback = new exposure off; active sessions drain naturally. Security rollback = secret rotation immediately."

### Q6 (MEDIUM) — Chrome FSA API compatibility
**Finding**: No deprecation signal found. `"showDirectoryPicker" in window` necessary but not sufficient — user denial, managed-browser enterprise policy, or user-activation failures can still fail.  
**Rec**: Keep Chrome/Edge smoke tests; monitor picker failures as operational noise.  
**Mitigation adopted**: Phase F smoke test (incognito = no managed policy) covers standard case. Enterprise managed-browser failures = acceptable operational noise below invariant threshold.

## Codex post-flip review (Phase I)

**실행 시각**: 2026-05-21T (Session 2, T+initial 데이터 기준)  
**판정**: PASS (nit-only) — T+1h 데이터 부재, smoke test 4/4 통과 기반 판정

**주의**: Phase I는 T+1h 메트릭 이후 실행 권장. 현재 T+initial(all-0) 기반으로 선행 판정.

### Q1 — 모니터링 데이터가 "ON-flip 정상 작동"의 충분 증거인가?

**판정**: 부분적 충분. Smoke test 4/4 (UX 2종 + HMAC 401 + env 우선) = 코드 레벨 정상 작동 직접 증거. 그러나 axis2.started=0은 실 사용자 파이프라인을 아직 검증하지 못함. **충분 증거 최소 조건**: axis2.started ≥ 1 AND failRate ≤ 20% (분모 ≥ 5). 현재 미충족 — T+1h 체크 필수.

**False positive 위험**: started=0이므로 카운터가 OK로 보이는 가짜 성공은 없음. 오히려 반대 위험(트래픽 부재로 메트릭 불확실).

### Q2 — T+15분 failRate 계산의 분모 < 10 misleading 가능성

**판정**: 해결됨. 분모=0이므로 failRate 자체가 정의 불가. 모니터링 표에 "N/A (분모<10) — noise" 명시. plan §Phase G에 "분모 < 10이면 noise로 처리" 이미 명시. **nit**: T+1h 판정 기준 재확인 필요 — 분모 ≥ 10이어야 failRate ≤ 20% 판정 유효.

### Q3 — 추가 invariant 누락 있는가?

**Finding (nit)**: 새 deployment가 진짜 새 빌드(새 commit hash가 아닌 재빌드)인지 확인 권장. EUAHZpz1Z deployment가 env=true를 inline한 fresh build인지 Vercel 배포 로그에서 "Use cache: false" 확인이 plan에 명시됐으나 실행 증거 미기재. 
**결론**: 기능적으로 smoke test #1이 이를 대리 검증 (env=true가 작동하지 않으면 "Auto-detect Burn Summary" 미표시). **severity: nit**.

### Q4 — 24h+ 정기 점검 cadence 미정 — 단발성 적절한가?

**Finding**: 본 plan은 T+1h까지를 scope으로 정의. 24h+ 장기 모니터링은 Non-scope으로 명시됨. 단발성 적절. 그러나 **T+1h에 axis2.started=0이면 성공조건 #5 미충족 → owner 판단 필요**: Axis 1 recruitment 캠페인 착수 또는 추가 노출 채널 확인.

**권고 (nit)**: T+1h 체크 결과에 따라 다음 사이클(axis1-recruitment-strategy.md)의 킥오프 여부 결정. 본 plan에 "T+1h 이후 owner 1회 점검 의무"를 명시 권장 → `status-2026-05-21.md`에 기재.
